# Optional scope — not part of the core MVP

Everything in this directory is deliberately separated from the core Glossa plan.

Rules:

- None of these features or services may block the core milestones.
- None are required login checkpoints for the core agent run.
- Do not provision paid optional services without explicit owner approval.
- Do not expand the relay architecture merely to accommodate a future feature.
- Revisit this directory only after `docs/09-acceptance-tests.md` passes or the owner explicitly changes scope.

Files:

- `FEATURES.md` — post-MVP product and architecture features
- `SERVICES.md` — nonessential Student Pack tools and alternative providers
- `TASKS.md` — optional backlog, ordered after core launch
- `LOGIN_CHECKPOINTS.md` — optional browser-login pauses
- `checkpoints.yaml` — machine-readable optional checkpoints
