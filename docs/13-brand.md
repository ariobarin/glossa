# Brand and naming guide

## Primary

**Glossa**

A concise name suggesting language, communication, and an interface between an agent and a local workspace.

## Correct usage

- Glossa
- glossa.sh
- `glossa login`
- `glossa status`
- `@ariobarin/glossa`
- Glossa relay
- Glossa worker

## Incorrect usage

- `glossa.sh login`
- Glossa.sh as the CLI name
- Veronica in current product UI
- unscoped `npm install glossa`

## Suggested copy

Headline:

> Let your agent work where your code already lives.

Subhead:

> Glossa securely connects an AI agent to a repository on your computer—without uploading the repo or configuring a VPS.

Install:

```bash
npm install --global @ariobarin/glossa
glossa login
```

## Core domains

- `glossa.sh` — existing marketing site; registered through Vercel Domains
- `mcp.glossa.sh` — Heroku relay, control API, and MCP endpoint

Do not move the apex site or registrar during the MVP. Other subdomains belong to the optional roadmap.
