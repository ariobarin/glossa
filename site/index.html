How it works

An execution bridge, not an agent.

  1. 1

    ChatGPT

    Owns the conversation, planning, and approvals.

  3. 2

    Managed relay

    Routes calls to your worker. Never stores files.

  5. 3

    Local worker

    Runs on your machine, in one folder.

Features

Containment without the ops.

No infrastructure to manage

No VPS, VPN, proxy, or database to run.

One exposed root

Scoped to one worktree or directory. Nothing broader.

Bounded commands

One command at a time, with firm timeouts.

Account tenancy

Your devices, workspaces, and jobs stay yours.

Quickstart

Three commands to a connected workspace.

  1. Install the CLI

    Private beta builds live on npm.

    npm install --global @ariobarin/glossa@beta

  2. Sign in

    Sign in through Auth0. Admission is explicit.

    glossa login

  3. Expose one workspace

    Run it in a worktree. Ctrl+C to stop.

    cd path/to/a/project
glossa

Then connect ChatGPT

Point a ChatGPT custom app at this endpoint. 

https://mcp.glossa.sh/mcp
